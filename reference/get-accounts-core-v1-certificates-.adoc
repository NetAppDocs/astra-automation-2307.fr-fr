= Récupérer un certificat
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/certificates/{certificate_id}`#

Renvoie la ressource API de certificat stockée dans le corps de la réponse JSON.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| identifiant_certificat | chaîne | chemin | Vrai  a| 
ID de la collection de certificats à lister

|===


== Réponse

[source]
----
Status: 200, Returns the stored Certificate API resource in the JSON response body.
----
[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-certificate"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| certUse | chaîne | Vrai  a| 
Chaîne JSON indiquant l'utilisation du certificat. Si non spécifié, la valeur par défaut est "rootCA". Les valeurs définies sont :

* « rootCA » - Le certificat est un certificat d'autorité de certification (CA) utilisé pour faire confiance à tout certificat signé par cette CA.
* « intermediateCA » - Le certificat est un certificat d’autorité de certification intermédiaire émis par une autorité de certification racine.




| certificat | chaîne | Vrai  a| 
Chaîne JSON contenant un certificat au format PKCS-11 encodé en base64 (PEM). Les valeurs définies sont :

* Conforme au schéma de données encodées en base64




| cn | chaîne | Vrai  a| 
Chaîne JSON contenant un nom commun extrait du certificat. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 511




| expiryTimestamp | chaîne | Vrai  a| 
Chaîne JSON contenant un horodatage indiquant la date d'expiration du certificat. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| isSelfSigned | chaîne | Vrai  a| 
Chaîne JSON indiquant si le certificat est auto-signé. Si cette information n'est pas fournie lors de la création, la valeur par défaut est "false". Lors de la mise à jour :

* Si une valeur est fournie, cette valeur sera utilisée.
* Si aucune valeur n'est fournie et que le champ « cert » est inclus dans la mise à jour, la valeur sera définie sur "false".
* Si aucune valeur n'est fournie et que le champ « cert » n'est pas inclus dans la mise à jour, la valeur existante sera conservée. Les valeurs définies sont :
* « true » - Le certificat est auto-signé.
* « false » - Le certificat n'est pas auto-signé.




| trustState | chaîne | Vrai  a| 
Chaîne JSON contenant l'une des valeurs énumérées indiquant si le certificat est approuvé. Les valeurs définies sont :

* « non fiable » - Le certificat n'est pas fiable.
* « trusted » - Le certificat est marqué comme fiable.
* « Expiré » - Le certificat a expiré, il n'est donc ni fiable ni non fiable.




| trustStateTransitions | tableau[link:#type_astra_stateTransition["type_astra_stateTransition"]] | Vrai  a| 
Tableau JSON d'objets JSON, chacun représentant l'ensemble des transitions d'état autorisées à partir d'un état défini. Chaque objet JSON contient les champs « from » et « to », où la valeur de « from » correspond au nom d'un état et celle de « to » à un tableau d'états de destination autorisés.



| trustStateDesired | chaîne | Faux  a| 
Chaîne JSON contenant une valeur indiquant si le certificat est approuvé. Si cette valeur n'est pas spécifiée lors de la création, la valeur par défaut est "approuvé". Les valeurs définies sont :

* « non fiable » - Le certificat n'est pas fiable.
* « trusted » - Le certificat est marqué comme fiable.




| trustStateDetails | tableau[link:#type_astra_stateDetail["type_astra_stateDetail"]] | Vrai  a| 
Tableau JSON d'objets JSON répertoriant des informations supplémentaires sur l'état de confiance du certificat. Si aucun détail n'est disponible, ce tableau sera vide.

* Nombre minimum d'articles : 0




| les métadonnées | link:#type_astra_metadata_update["type_astra_metadata_update"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette. Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.
