= Lister toutes les licences
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/licenses`#

Renvoie un tableau JSON contenant toutes les ressources correspondantes de la collection. Si des champs individuels sont spécifiés dans la requête, le tableau JSON items contiendra les champs demandés pour chaque ressource correspondante dans l'ordre indiqué.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| inclure | chaîne | requête | Faux  a| 
Indique les champs qui doivent être renvoyés lors de l’énumération d’une collection.

Exemples d'URL : « include=id », « include=id,name »

Exemples OpenAPI : « id »

* Exemple : identifiant, produit




| limite | chaîne | requête | Faux  a| 
Indique le nombre maximal de ressources à renvoyer lors de l’énumération d’une collection.

Exemples d'URL : « limit=2 »

Exemples OpenAPI : « 2 »



| filtre | chaîne | requête | Faux  a| 
Indique les champs qui doivent être mis en correspondance pour renvoyer une ressource lors du listage d'une collection.

Les opérateurs pris en charge sont :

* eq - Égalité
* lt/gt - Inférieur à/Supérieur à
* LTE/GTE - Inférieur ou égal à/Supérieur ou égal à


Exemple d'URL : « filter=field%20eq%20%27value%27 »

Exemples OpenAPI : « field eq 'value' »

|===

corps de réponse license_1.0_get_response_body

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-license"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| allocation | chaîne | Faux  a| 
Chaîne JSON contenant un identifiant indiquant à quoi la licence est attribuée. Présent uniquement lorsque la licence est attribuée. Pour les licences Astra Control Center, ce champ doit être renseigné avec l'identifiant du compte et doit correspondre à l'identifiant du compte spécifié dans l'URI POST. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| ID hôte | chaîne | Faux  a| 
Chaîne JSON contenant un identifiant d'hôte auquel une licence est associée. Présente uniquement si la licence est liée à un hôte. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| identifiant d’identification de l’appareil | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant du deviceCredential associé à l'API Credential. Présente uniquement pour les licences Astra Data Store. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| isEvaluation | chaîne | Vrai  a| 
Chaîne JSON indiquant si la licence est une licence d'évaluation. Les valeurs définies sont :

* "true"
* "false"




| licenseProtocol | chaîne | Vrai  a| 
Chaîne JSON contenant le protocole de licence extrait de la licence. Les valeurs définies sont :

* Contenu de chaîne non validé par le schéma




| licenseText | chaîne | Vrai  a| 
Chaîne JSON contenant une représentation encodée en base64 du JSON de la licence, telle qu'obtenue auprès du système de licences NetApp. Cette chaîne doit être fournie dans l'ordre d'octets d'origine pour permettre la vérification de la signature. Les valeurs définies sont :

* Conforme au schéma de données encodées en base64




| validFromTimestamp | chaîne | Vrai  a| 
Chaîne JSON contenant un horodatage indiquant quand la licence devient valide. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| validUntilTimestamp | chaîne | Vrai  a| 
Chaîne JSON contenant un horodatage indiquant quand la licence n'est plus valide. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| produit | chaîne | Vrai  a| 
Chaîne JSON contenant la chaîne de produit extraite de la licence. Les valeurs définies sont :

* Contenu de chaîne non validé par le schéma




| productVersion | chaîne | Vrai  a| 
Chaîne JSON contenant la chaîne de version extraite de la licence. Les valeurs définies sont :

* Contenu de chaîne non validé par le schéma




| productSN | chaîne | Vrai  a| 
Chaîne JSON contenant le numéro de série extrait de la licence. Les valeurs définies sont :

* Contenu de chaîne non validé par le schéma




| caractéristiques | chaîne | Vrai  a| 
Chaîne JSON contenant la valeur du package extraite de la licence. Les valeurs définies sont :

* Contenu de chaîne non validé par le schéma




| capacité | chaîne | Vrai  a| 
Chaîne JSON contenant la capacité extraite de la licence.



| capacity2 | chaîne | Vrai  a| 
Chaîne JSON contenant la capacité secondaire extraite de la licence.



| modules complémentaires | tableau[link:#addons["modules complémentaires"]] | Faux  a| 
Tableau JSON d'objets JSON contenant la liste des add-ons de la licence. Un add-on correspond à une modification de la licence pouvant intervenir à une date ultérieure.

Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.
