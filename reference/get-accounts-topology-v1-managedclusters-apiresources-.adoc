= Récupérer une ressource d'API Kubernetes
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/managedClusters/{managedCluster_id}/apiResources/{apiResource_id}`#

Renvoie la ressource API spécifique dans le corps de la réponse JSON.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| id_cluster_géré | chaîne | chemin | Vrai  a| 
ID de la ressource managedCluster conteneur



| apiResource_id | chaîne | chemin | Vrai  a| 
ID de la collection apiResource à lister

|===


== Réponse

[source]
----
Status: 200, Returns the specific API resource in the JSON response body.
----
[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-apiResource"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| apiVersion | chaîne | Vrai  a| 
Chaîne JSON représentant la version de l'API de la ressource, telle que définie dans ou par Kubernetes. APIVersion définit le schéma versionné pour cette représentation d'objet. Les serveurs doivent convertir les schémas reconnus vers la dernière valeur interne et peuvent rejeter les valeurs non reconnues. Plus d'informations : https://git.k8s.io/community/contributors/devel/sig-architecture/api-conventions.md#resources[]



| type | chaîne | Vrai  a| 
Chaîne JSON représentant le type de la ressource, tel que défini dans ou par Kubernetes. Le champ kind utilise la casse chameau et contient une valeur de chaîne représentant la ressource REST que cet objet représente. Les serveurs peuvent l'inférer à partir du point de terminaison auquel le client soumet les demandes. Il ne peut pas être mis à jour. Plus d'informations : https://git.k8s.io/community/contributors/devel/sig-architecture/api-conventions.md#types-kinds[]



| les métadonnées | link:#type_astra_metadata["type_astra_metadata"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra


|===

  }
}
----
====
