= Récupérer une tâche
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/tasks/{task_id}`#

Renvoie la ressource d'API Task stockée dans le corps de la réponse JSON.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| identifiant_de_tâche | chaîne | chemin | Vrai  a| 
ID de la collection de tâches à lister

|===


== Réponse

[source]
----
Status: 200, Returns the stored Task API resource in the JSON response body.
----
[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-task"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Vrai  a| 
Chaîne JSON contenant un nom hiérarchique pour la tâche en notation point (.). Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 127
* L'expression régulière définie est : "^(([a-z])_(\.))_([a-z])+$"




| résumé | chaîne | Vrai  a| 
Chaîne JSON contenant le résumé court de la tâche. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 63




| description | chaîne | Vrai  a| 
Chaîne JSON contenant une description de la tâche. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 511




| service | chaîne | Faux  a| 
Chaîne JSON contenant le nom du service qui gère la tâche. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 31




| parentTaskID | chaîne | Faux  a| 
Chaîne JSON contenant un identifiant unique global de la tâche parente de la tâche. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| ID de l'utilisateur | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant utilisateur associé à une tâche. Uniquement incluse pour les tâches associées à des actions initiées par l'utilisateur. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| identifiant de ressource | chaîne | Vrai  a| 
Chaîne JSON contenant un identifiant unique global de la ressource associée à la tâche. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| URI de ressource | chaîne | Vrai  a| 
Chaîne JSON contenant l'URI associée à la tâche. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 4095




| URI de la collection de ressources | tableau[chaîne] | Vrai  a| 
Tableau JSON de chaînes JSON, chacune indiquant l'URL d'accès complète de la ressource associée à la tâche. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 4095




| État | chaîne | Vrai  a| 
Chaîne JSON contenant une valeur indiquant l'état opérationnel de la tâche. Les valeurs définies sont :

* "notStarted" - La tâche n'a pas commencé à s'exécuter.
* "en cours d'exécution" - La tâche est en cours d'exécution.
* « completed » - La tâche est terminée.
* "pausing" - La tâche a été mise en pause.
* "en pause" - La tâche est en pause.
* « Annulation » - L'annulation de la tâche a été demandée.
* « Annulé » - La tâche est annulée.
* « failed » - La tâche a échoué.
* Longueur minimale : 1
* Longueur maximale : 31




| stateTransitions | tableau[link:#type_astra_stateTransition["type_astra_stateTransition"]] | Vrai  a| 
Tableau JSON d'objets JSON, chacun représentant l'ensemble des transitions d'état autorisées à partir d'un état défini. Chaque objet JSON contient les champs « from » et « to », où la valeur de « from » correspond au nom d'un état et celle de « to » à un tableau d'états de destination autorisés.

* Longueur minimale : 1
* Longueur maximale : 31 Par exemple : «stateTransitions» : [ { "from": "running", "to": ["paused", "cancelled"] }, { "from": "paused", "to": ["running", "cancelled"] }, ]




| stateDetails | tableau[link:#type_astra_stateDetail["type_astra_stateDetail"]] | Vrai  a| 
Tableau JSON d'objets JSON contenant des informations supplémentaires sur l'état actuel. Ce champ indique lorsqu'il existe des conditions ou des erreurs qui empêchent l'état réel de correspondre à l'état souhaité. Si aucun détail n'est disponible, ce tableau sera vide.

* Nombre minimum d'articles : 0




| orderHint | nombre | Faux  a| 
Numéro JSON représentant l'ordre des sous-tâches. Ce champ peut être utilisé par le client pour trier les sous-tâches de la plus petite à la plus grande.



| percentDone | nombre | Faux  a| 
Nombre JSON contenant le pourcentage d'avancement d'une tâche en cours. Lorsque la tâche est terminée, ce nombre est défini sur "100".



| startTime | chaîne | Faux  a| 
Chaîne JSON contenant l'horodatage indiquant quand la tâche a commencé. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| endTime | chaîne | Faux  a| 
Chaîne JSON contenant l'horodatage indiquant quand la tâche s'est terminée. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| cancelTime | chaîne | Faux  a| 
Chaîne JSON contenant l'horodatage indiquant le moment où la tâche a été annulée. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| les métadonnées | link:#type_astra_metadata["type_astra_metadata"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra
