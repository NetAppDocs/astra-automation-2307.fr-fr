= Modifier un compartiment
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-put]#PUT# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/buckets/{bucket_id}`#

Remplace l'intégralité de la ressource stockée par le corps de la requête JSON fourni, tout en préservant les valeurs clés non modifiables par l'utilisateur.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| bucket_id | chaîne | chemin | Vrai  a| 
ID de la collection de compartiments à lister

|===


== Corps de la requête

Remplace l'intégralité de la ressource stockée par le corps de la requête JSON fourni, tout en préservant les valeurs clés non modifiables par l'utilisateur.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-bucket"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"
* "1,2"




| id | chaîne | Faux  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Faux  a| 
Chaîne JSON contenant un nom attribué par l'utilisateur au compartiment. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 256




| identifiant | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant des identifiants utilisés pour accéder au compartiment. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| État | chaîne | Faux  a| 
Chaîne JSON contenant une valeur indiquant l'état opérationnel du compartiment. Les valeurs définies sont :

* « pending » - Le compartiment est programmé pour être découvert
* « disponible » - Le seau est disponible pour utilisation
* « défaillant » - Le bucket est dans un état de défaillance permanent
* « supprimé » - Le compartiment n'est actuellement pas accessible
* « unknown » - L'état du compartiment ne peut être déterminé




| stateDetails | tableau[link:#type_astra_stateDetail["type_astra_stateDetail"]] | Faux  a| 
Tableau JSON d'objets JSON contenant des informations supplémentaires sur la raison pour laquelle « state » est passé à « removed » ou « failed ». Si aucun détail n'est disponible, ce tableau sera vide.

* Nombre minimum d’articles : 0




| fournisseur | chaîne | Faux  a| 
Chaîne JSON indiquant le protocole de compartiment pouvant être utilisé pour accéder au compartiment. Les valeurs définies sont :

* "ontap-s3"
* "storagegrid-s3"
* "generic-s3"
* "gcp"
* "azure"
* "aws"




| bucketParameters | link:#bucketParameters["bucketParameters"] | Faux  a| 
Objet JSON contenant les informations spécifiques au fournisseur de compartiment nécessaires pour accéder au compartiment. Les valeurs spécifiques de ce compartiment varient selon le fournisseur.



| les métadonnées | link:#type_astra_metadata_update["type_astra_metadata_update"] | Faux  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette. Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.


|===
.Exemple de requête
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-bucket",
  "version": "1.2",
  "name": "New Bucket Name"
}
----
====
