= Créer un nouvel instantané d'application pour une application
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-post]#POST# [.api-doc-code-block]#`/accounts/{account_id}/k8s/v1/apps/{app_id}/appSnaps`#

Indique les valeurs souhaitées pour la ressource d'API Application Snapshot à créer.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| app_id | chaîne | chemin | Vrai  a| 
ID de la ressource d'application contenant

|===


== Corps de la requête

Indique les valeurs souhaitées pour la ressource d'API Application Snapshot à créer.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-appSnap"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"
* "1,2"




| nom | chaîne | Faux  a| 
Chaîne JSON contenant un nom défini par l'utilisateur pour l'instantané. Si aucun nom n'est spécifié, ou lorsque les instantanés sont déclenchés par une planification, le système attribue un nom à l'instantané. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63
* Conforme au schéma d'étiquette DNS 1123




| les métadonnées | link:#type_astra_metadata["type_astra_metadata"] | Faux  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette.


|===
.Exemple de requête
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-appSnap",
  "version": "1.2",
  "name": "app-name-245"
}
----
====


== Réponse

[source]
----
Status: 201, Returns the newly created appSnap resource in the JSON response body.
----
[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-appSnap"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"
* "1,2"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Vrai  a| 
Chaîne JSON contenant un nom défini par l'utilisateur pour l'instantané. Si aucun nom n'est spécifié, ou lorsque les instantanés sont déclenchés par une planification, le système attribue un nom à l'instantané. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63
* Conforme au schéma d'étiquette DNS 1123




| ID de planification | chaîne | Faux  a| 
Chaîne JSON contenant un ID de planification facultatif indiquant que l'instantané a été déclenché par une planification et précisant quelle planification a déclenché l'instantané. Présent uniquement pour les instantanés déclenchés par une planification. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| snapshotAppAsset | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant de la ressource d'application correspondant à l'instantané. Ce champ sera renvoyé lorsque l'état de l'instantané de l'application est « completed ». Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| État | chaîne | Vrai  a| 
Chaîne JSON contenant une valeur indiquant l'état opérationnel de l'instantané. Les valeurs définies sont :

* « En attente » - L’instantané de l’application est programmé pour être découvert
* « discovering » - L'instantané de l'application est en cours de découverte
* « running » - L’instantané de l’application fonctionne normalement
* « completed » - L'instantané de l'application est terminé
* « Échec » - L’instantané de l’application est dans un état d’échec permanent
* « Supprimé » - L'instantané de l'application n'est actuellement pas accessible
* « unknown » - L'état de l'instantané de l'application ne peut être déterminé




| stateUnready | tableau[chaîne] | Vrai  a| 
Tableau JSON de chaînes JSON, chacune indiquant la raison pour laquelle un instantané d'application ne peut pas passer à l'état « en cours d'exécution » ou pourquoi il est passé à l'état « en échec ». Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 127




| hookState | chaîne | Faux  a| 
Chaîne JSON contenant une valeur indiquant l'état d'un ou plusieurs hooks d'exécution exécutés dans le cadre d'un instantané. Les valeurs définies sont :

* « failed » - Un ou plusieurs hooks d'exécution ont échoué
* « succès » - Il y avait zéro ou plusieurs crochets applicables et tous ont réussi




| hookStateDetails | tableau[link:#type_astra_stateDetail["type_astra_stateDetail"]] | Faux  a| 
Tableau JSON d'objets JSON contenant des détails supplémentaires sur la raison pour laquelle «hookState» est passé à «failed». Si aucun détail n'est disponible, ce tableau sera vide.

* Nombre minimum d’articles : 0




| les métadonnées | link:#type_astra_metadata["type_astra_metadata"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette.


|===

  "name": "app-name-245",
  "scheduleID": "1fe83650-9b3c-41ef-bb25-a2910c19c33e",
  "state": "pending",
  "stateUnready": [],

  }
}
----
