= Récupérer une source de hook
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/hookSources/{hookSource_id}`#

Renvoie la ressource API Hook Source stockée dans le corps de la réponse JSON.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| hookSource_id | chaîne | chemin | Vrai  a| 
ID de la collection hookSource à lister

|===


== Réponse

[source]
----
Status: 200, Returns the stored Hook Source API resource in the JSON response body.
----
[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-hookSource"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Vrai  a| 
Chaîne JSON contenant un nom spécifié par l'utilisateur pour la ressource. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63 Le nom doit être unique parmi toutes les ressources sources du hook.




| privé | chaîne | Vrai  a| 
Chaîne JSON indiquant si le champ source sera exposé dans les API ou restera intentionnellement vide. Les valeurs définies sont :

* « true » - La valeur source est privée et ne sera pas exposée via l'API.
* "false" - La valeur source n'est pas privée et sera exposée via l'API.




| préchargé | chaîne | Vrai  a| 
Chaîne JSON indiquant si la source a été préchargée. Les valeurs définies sont :

* "true" - La source a été préchargée.
* « false » : la source n’a pas été préchargée. Les ressources sources des hooks préchargées sont en lecture seule.




| sourceType | chaîne | Vrai  a| 
Chaîne JSON définissant le type de source du hook. Les valeurs définies sont :

* « script » - Script shell. Le code source en texte brut ne doit pas contenir de données binaires ni de caractères de retour chariot, sous peine de générer une erreur.




| source | chaîne | Faux  a| 
Chaîne JSON contenant le script à exécuter au format base64. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 131072
* Conforme au schéma de données encodées en base64 La taille maximale d'une valeur source est de 128 KiB lorsqu'elle est encodée en base64.




| sourceMD5Checksum | chaîne | Vrai  a| 
Chaîne JSON contenant la somme de contrôle MD5 calculée pour la valeur de la clé « source ». Les valeurs définies sont :

* L'expression régulière définie est : "^([0-9a-fA-F]){32}$"




| description | chaîne | Faux  a| 
Chaîne JSON contenant une description de la source du hook spécifiée par l'utilisateur. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 511




| les métadonnées | link:#type_astra_metadata_update["type_astra_metadata_update"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette. Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.
