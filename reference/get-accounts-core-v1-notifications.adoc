= Lister toutes les notifications
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/notifications`#

Renvoie un tableau JSON contenant toutes les ressources correspondantes de la collection. Si des champs individuels sont spécifiés dans la requête, le tableau JSON items contiendra les champs demandés pour chaque ressource correspondante dans l'ordre indiqué.

== Définitions

.Voir les définitions

[role="api-collapsible-fifth-title"]
données

Objet JSON contenant des informations supplémentaires spécifiques à l'événement.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| ttl | nombre | Faux  a| 
Nombre JSON indiquant le nombre de secondes (depuis le eventTime) pendant lesquelles l'événement sera conservé. Ce champ est uniquement inclus lorsque l'événement est automatiquement supprimé après le nombre de secondes spécifié. Lors de la suppression, si l'événement a une destination de notification, toutes les notifications associées sont également supprimées. Lors de la suppression, si l'événement a une destination de bannière, la bannière est également supprimée. L'absence de ce champ (ou une valeur de « 0 ») signifie qu'un événement ne sera pas automatiquement supprimé.



| isAcknowledgeable | chaîne | Faux  a| 
Chaîne JSON indiquant si la bannière peut être masquée. Les valeurs définies sont :

* "true" - La bannière est escamotable
* « false » - La bannière ne peut pas être supprimée

|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-notification"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"
* "1,2"
* "1,3"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Vrai  a| 
Chaîne JSON contenant un nom hiérarchique pour l'événement en notation point (.). Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 127
* L'expression régulière définie est : "^(([a-z])_(\.))_([a-z])+$"




| sequenceCount | nombre | Vrai  a| 
Numéro JSON contenant un compteur de séquence. Le numéro est incrémenté par le système à chaque fois qu’un nouvel événement est reçu.



| résumé | chaîne | Vrai  a| 
Chaîne JSON contenant le résumé abrégé (moins de 40 caractères) de l'événement. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 79




| eventTime | chaîne | Vrai  a| 
Chaîne JSON contenant un horodatage indiquant le moment où un événement s'est produit. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601




| source | chaîne | Vrai  a| 
Chaîne JSON contenant le nom en minuscules du service générant un événement. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 19
* L'expression régulière définie est : "^([a-z-])*$"




| identifiant de ressource | chaîne | Vrai  a| 
Chaîne JSON contenant l'identifiant de la ressource associée à un événement. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| identifiants de ressources supplémentaires | tableau[chaîne] | Vrai  a| 
Tableau JSON de chaînes de caractères JSON, chacune contenant un identifiant de ressource supplémentaire associée à l'événement. Bien que le champ « resourceID » contienne la ressource principale associée à cet événement, ce champ contient n'importe quel nombre d'identifiants de ressources supplémentaires à associer aux mêmes événements. Si aucune ressource supplémentaire n'est associée à l'événement, le tableau sera vide. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| resourceType | chaîne | Vrai  a| 
Chaîne JSON contenant le type de média de la ressource associée à un événement. Les valeurs définies sont :

* Longueur minimale : 4
* Longueur maximale : 79
* L'expression régulière définie est : "^application\/astra-([a-zA-Z])+$"




| identifiant de corrélation | chaîne | Vrai  a| 
Chaîne JSON contenant un identifiant de corrélation UUIDv4 permettant de regrouper les événements associés. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| gravité | chaîne | Vrai  a| 
Chaîne JSON contenant l'une des valeurs suivantes indiquant la gravité d'un événement. Les valeurs définies sont :

* « Résolu » : indique qu’un événement précédemment signalé décrivant une condition n’est plus actif. Un événement dont la gravité est « cleared » conservera le même résumé et le même identifiant de corrélation pour permettre son association.
* « indeterminate » – Indique que la gravité ne peut pas être déterminée.
* « informationnel » - Indique un changement d'état à la fois attendu et normal (non défini dans X.733).
* « warning » - Indique la détection d’un défaut potentiel ou imminent affectant le service, avant que des effets significatifs ne se fassent sentir.
* « critical » : Indique qu’un incident affectant le service s’est produit et qu’une action corrective immédiate est requise. Les définitions de « cleared », « indeterminate », « warning » et « critical » sont basées sur les définitions de la clause 8.1.2.3 du CCITT X.733.




| classe | chaîne | Vrai  a| 
Chaîne JSON contenant l'une des valeurs suivantes indiquant la classe de l'événement. Les valeurs définies sont :

* « système » - L'événement est lié à un événement ou une requête interne.
* « utilisateur » - L'événement est lié à une action ou une requête initiée par l'utilisateur.
* « sécurité » - L'événement est lié à un problème de sécurité.




| description | chaîne | Vrai  a| 
Chaîne JSON contenant un texte descriptif long concernant l'événement. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 1023




| URL de description | chaîne | Faux  a| 
Chaîne JSON contenant l'URL qui active et lie un bouton de la bannière à la documentation externe. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 4095




| correctiveAction | chaîne | Faux  a| 
Chaîne JSON contenant un texte descriptif détaillé sur les mesures correctives à prendre suite à l'événement. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 1023




| URL de l'action corrective | chaîne | Faux  a| 
Chaîne JSON contenant l'URL qui relie un bouton de la bannière à une page de l'interface utilisateur permettant de résoudre le problème. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 4095




| visibilité | tableau[chaîne] | Faux  a| 
Tableau JSON de chaînes JSON, chacune indiquant un rôle RBAC autorisé à consulter l'événement. Les rôles sont définis par un OU logique. Facultatif. Si omis, un utilisateur disposant de n'importe quel rôle RBAC peut consulter l'événement. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| destinations | tableau[chaîne] | Faux  a| 
Tableau JSON de chaînes JSON, chacune indiquant une destination supplémentaire pour l'événement. Ces chaînes permettent à un service de fournir des indications sur l'endroit où un événement doit être routé en plus du journal d'activité du service Activity. Facultatif. Les valeurs définies sont :

* « notification » - L'événement est marqué comme une notification et sera accessible via le point de terminaison de l'API de notifications.
* "bannière" - L'événement est marqué comme une bannière.
* « support » - L'événement est marqué comme un événement lié au support.




| URI de ressource | chaîne | Faux  a| 
Chaîne JSON contenant l'URI associée à l'événement. Uniquement incluse pour les événements liés aux opérations d'API. Les valeurs définies sont :

* Longueur minimale : 3
* Longueur maximale : 4095




| URL de la collection de ressources | tableau[chaîne] | Faux  a| 
Tableau JSON de chaînes JSON, chacune indiquant l'URL d'accès complète de la ressource associée à l'événement. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 1023




| resourceMethod | chaîne | Faux  a| 
Chaîne JSON contenant la méthode HTTP appelée. Incluse uniquement pour les événements liés aux opérations d'API. Les valeurs définies sont :

* "options" - Opération de l'API OPTIONS
* "post" - Opération API POST
* "get" - Opération API GET
* « put » - Opération d'API PUT
* « supprimer » - Opération d'API DELETE




| resourceMethodResult | chaîne | Faux  a| 
Chaîne JSON contenant le résultat de la méthode HTTP appelée. Uniquement incluse pour les événements liés aux opérations de l'API. Les valeurs définies sont :

* L'expression régulière définie est : "^[1-5][0-9]{2}$"




| ID de l'utilisateur | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant utilisateur associé à un événement. Uniquement inclus pour les événements liés à des actions initiées par l'utilisateur. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| identifiant de compte | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant du compte associé à un événement. Uniquement inclus pour les événements associés à un compte. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra

Objet JSON contenant des informations supplémentaires spécifiques à l'événement.

Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :
