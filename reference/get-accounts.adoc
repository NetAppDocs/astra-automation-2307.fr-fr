= Lister tous les comptes
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts`#

Renvoie un tableau JSON contenant toutes les ressources correspondantes de la collection. Si des champs individuels sont spécifiés dans la requête, le tableau JSON items contiendra les champs demandés pour chaque ressource correspondante dans l'ordre indiqué.

|===

Exemples OpenAPI : « id »

Indique le nombre maximal de ressources à renvoyer lors de l’énumération d’une collection.

[role="api-collapsible-fifth-title"]
postalAddress

Objet JSON contenant une collection de chaînes JSON spécifiant une adresse. Si cette clé n'existe pas, cela indique qu'aucune adresse n'a été fournie. Si cette clé existe, les six chaînes JSON qu'elle contient seront présentes. Si vous ne fournissez rien, la chaîne sera vide.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| addressCountry | chaîne | Vrai  a| 
Chaîne JSON contenant le nom ou le code pays associé à l'adresse postale, conformément à la norme ISO 3166 Alpha-2. Les valeurs définies sont :

* Longueur minimale : 2
* Longueur maximale : 2




| addressLocality | chaîne | Vrai  a| 
Chaîne JSON contenant la localité (par exemple, la ville) associée à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| addressRegion | chaîne | Vrai  a| 
Chaîne JSON contenant la région (par exemple, l'État) associée à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| postalCode | chaîne | Vrai  a| 
Chaîne JSON contenant le code postal associé à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 31




| streetAddress1 | chaîne | Vrai  a| 
Chaîne JSON contenant l'adresse de rue associée à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| streetAddress2 | chaîne | Faux  a| 
Chaîne JSON contenant des informations complémentaires facultatives telles que l'adresse de rue, la boîte postale, et plus encore. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63


|===
[role="api-collapsible-fifth-title"]
accountContact

Objet JSON contenant les informations de contact de l'utilisateur permettant de créer un utilisateur propriétaire lors de l'activation du compte.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| firstName | chaîne | Vrai  a| 
Chaîne JSON contenant le prénom long spécifié par l'utilisateur. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63
* Cette chaîne de caractères est validée en interne afin de prévenir les attaques XSS, Unicode, de traversée de répertoires et d'injection SQL.




| lastName | chaîne | Vrai  a| 
Chaîne JSON contenant un nom de famille long spécifié par l'utilisateur. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63
* Cette chaîne de caractères est validée en interne afin de prévenir les attaques XSS, Unicode, de traversée de répertoires et d'injection SQL.




| companyName | chaîne | Faux  a| 
Chaîne JSON contenant le nom de l'entreprise associée à l'utilisateur. Si cette clé n'existe pas, cela indique qu'un nom d'entreprise n'a pas été fourni. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63
* Cette chaîne de caractères est validée en interne afin de prévenir les attaques XSS, Unicode, de traversée de répertoires et d'injection SQL.




| e-mail | chaîne | Vrai  a| 
Chaînes JSON contenant une adresse e-mail spécifiée par l'utilisateur. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| téléphone | chaîne | Faux  a| 
Chaîne JSON contenant le numéro de téléphone de l'utilisateur. Si cette clé n'existe pas, cela indique qu'un numéro de téléphone n'a pas été fourni. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 31

Type de média de la ressource. Les valeurs définies sont :

| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Vrai  a| 
Chaîne JSON contenant un nom pour le compte. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63
* Cette chaîne de caractères est validée en interne afin de prévenir les attaques XSS, Unicode, de traversée de répertoires et d'injection SQL.




| État | chaîne | Vrai  a| 
Chaîne JSON contenant une valeur indiquant l'état opérationnel du compte. Les valeurs définies sont :

* « pending » - Les modifications ne peuvent être apportées qu'aux ressources utilisateur et de rôle du compte (valeur par défaut)
* « actif » : des modifications peuvent être apportées à n’importe quelle ressource du compte
* "deletePending" - Les utilisateurs ne peuvent pas se connecter, les opérations de protection des données sont suspendues, les frais ne sont plus facturés




| isEnabled | chaîne | Vrai  a| 
Chaîne JSON indiquant que le compte est activé. Les valeurs définies sont :

* "false" - Les utilisateurs ne sont pas autorisés à utiliser ce compte
* « true » - Les utilisateurs sont autorisés à utiliser ce compte




| enabledTimestamp | chaîne | Faux  a| 
Chaîne JSON contenant l'horodatage indiquant quand le compte a été activé pour la dernière fois. Si le compte n'a jamais été activé, ce champ est absent. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601

Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.
