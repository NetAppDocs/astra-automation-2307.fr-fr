= Créer un nouvel abonnement
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-post]#POST# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/subscriptions`#

Indique les valeurs souhaitées pour la ressource Subscription API à créer.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}


|===


== Corps de la requête

Indique les valeurs souhaitées pour la ressource Subscription API à créer.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-subscription"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"
* "1,2"




| ID de profil client | chaîne | Faux  a| 
La chaîne JSON contenant l’identifiant du profil client à partir de  http://authorize.net["Authorize.net"], qui a été invoqué par l’interface utilisateur, n’est pas définie pour le délai de grâce. Renvoie une chaîne vide lors de l’opération de lecture, si elle n’est pas définie. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63




| paymentFirstName | chaîne | Faux  a| 
Chaîne JSON contenant un prénom complet spécifié par l'utilisateur, utilisé pour la facturation. Spécifié uniquement lorsque l'abonnement est refacturé. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| paymentLastName | chaîne | Faux  a| 
Chaîne JSON contenant un nom de famille complet spécifié par l'utilisateur, utilisé pour la facturation. Spécifié uniquement lorsque l'abonnement est refacturé. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63




| paymentAddress | link:#paymentAddress["paymentAddress"] | Faux  a| 
Objet JSON contenant une collection de chaînes JSON spécifiant une adresse de facturation. Spécifié uniquement lorsque l'abonnement est refacturé. Si cette clé existe, les six chaînes JSON qu'elle contient seront présentes. Si rien n'a été fourni par l'utilisateur, la chaîne sera vide.



| ID de profil de paiement | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant du profil de paiement de http://Authorize.net["Authorize.net"] qui a été invoqué par l'interface utilisateur, non défini pour la période d'essai. Renvoie une chaîne vide lors de l'opération de lecture si non défini. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63




| paymentExpiry | chaîne | Faux  a| 
Chaîne JSON contenant un horodatage indiquant la date d'expiration du mode de paiement. Cette information est obtenue auprès d'authorize.net(http://authorize.net[] et définie par l'interface utilisateur. Les valeurs définies sont :

* Conforme au schéma de date et d'heure ISO-8601 Ce champ n'est pas renvoyé pour les périodes d'essai.




| marketplace | chaîne | Faux  a| 
Chaîne JSON indiquant la marketplace du compte sur laquelle les paiements seront enregistrés. Valeurs définies :

* "netapp"
* "azure"
* "aws"
* "gcp"




| termes | chaîne | Vrai  a| 
Chaîne JSON indiquant le niveau actuel du compte. Les valeurs définies sont :

* "essai"
* "payé"




| les métadonnées | link:#type_astra_metadata_update["type_astra_metadata_update"] | Faux  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette. Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.


|===
.Exemple de requête
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-subscription",
  "version": "1.2",
  "terms": "trial"
}
----
====

== Définitions

[role="api-collapsible-fifth-title"]
paymentAddress

Objet JSON contenant une collection de chaînes JSON spécifiant une adresse de facturation. Spécifié uniquement lorsque l'abonnement est refacturé. Si cette clé existe, les six chaînes JSON qu'elle contient seront présentes. Si rien n'a été fourni par l'utilisateur, la chaîne sera vide.

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| addressCountry | chaîne | Vrai  a| 
Chaîne JSON contenant le nom ou le code pays associé à l'adresse postale, conformément à la norme ISO 3166 Alpha-2. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 2




| addressLocality | chaîne | Vrai  a| 
Chaîne JSON contenant la localité (par exemple, la ville) associée à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63




| addressRegion | chaîne | Vrai  a| 
Chaîne JSON contenant la région (par exemple, l'État) associée à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63




| postalCode | chaîne | Vrai  a| 
Chaîne JSON contenant le code postal associé à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63




| streetAddress1 | chaîne | Vrai  a| 
Chaîne JSON contenant l'adresse de rue associée à l'adresse postale. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63




| streetAddress2 | chaîne | Faux  a| 
Chaîne JSON contenant des informations complémentaires facultatives telles que l'adresse de rue, la boîte postale, et plus encore. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 63
