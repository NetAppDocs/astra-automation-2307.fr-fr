= Lister tous les hooks d'exécution
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/executionHooks`#

Renvoie un tableau JSON contenant toutes les ressources correspondantes de la collection. Si des champs individuels sont spécifiés dans la requête, le tableau JSON items contiendra les champs demandés pour chaque ressource correspondante dans l'ordre indiqué.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| inclure | chaîne | requête | Faux  a| 
Indique les champs qui doivent être renvoyés lors de l’énumération d’une collection.

Exemples d'URL : « include=id », « include=id,name »

Exemples OpenAPI : « id »



| limite | chaîne | requête | Faux  a| 
Indique le nombre maximal de ressources à renvoyer lors de l’énumération d’une collection.

Exemples d'URL : « limit=2 »

Exemples OpenAPI : « 2 »



| filtre | chaîne | requête | Faux  a| 
Indique les champs qui doivent être mis en correspondance pour renvoyer une ressource lors du listage d'une collection.

Les opérateurs pris en charge sont :

* eq - Égalité
* lt/gt - Inférieur à/Supérieur à
* LTE/GTE - Inférieur ou égal à/Supérieur ou égal à


Exemple d'URL : « filter=field%20eq%20%27value%27 »

Exemples OpenAPI : « field eq 'value' »

|===

corps de réponse executionHook_1.3_get_response_body

[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-executionHook"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"
* "1,1"
* "1,2"
* "1,3"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| nom | chaîne | Vrai  a| 
Chaîne JSON contenant un nom spécifié par l'utilisateur pour la ressource. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 63 Le nom doit être unique pour toutes les ressources de hook d’exécution.




| hookType | chaîne | Vrai  a| 
Chaîne JSON définissant le type de hook. Les valeurs définies sont :

* "netapp" - Hook d'exécution fourni par NetApp.
* « custom » - lié à un ID d’application spécifique. Les ressources du hook d’exécution avec hookType « netapp » sont en lecture seule.




| matchingCriteria | tableau[link:#matchingCriteria["matchingCriteria"]] | Vrai  a| 
Tableau JSON d'objets JSON décrivant les critères de correspondance pour le hook d'exécution. Lorsque plusieurs matchingCriteria sont spécifiés, un ET logique est appliqué aux critères. Les valeurs définies sont :

* Nombre minimum d’articles : 0
* Nombre maximum d'articles : 10




| matchingContainers | tableau[link:#matchingContainers["matchingContainers"]] | Faux  a| 
Tableau JSON d'objets JSON pour chaque conteneur correspondant aux critères de correspondance configurés au sein de l'application définie.



| matchingImages | tableau[chaîne] | Faux  a| 
Tableau JSON de chaînes JSON pour les images conteneurs correspondantes. Les valeurs définies sont :

* Nombre minimum d’articles : 0
* Nombre maximum d'articles : 4095
* Longueur minimale : 1
* Longueur maximale : 255




| action | chaîne | Vrai  a| 
Chaîne JSON indiquant l'action que le hook d'exécution lancera si les critères de correspondance sont remplis. Les valeurs définies sont :

* "instantané"
* "sauvegarde"
* "restaurer"
* « basculement »




| scène | chaîne | Vrai  a| 
Chaîne JSON indiquant l'étape de l'action à laquelle le hook sera exécuté. Les valeurs définies sont :

* "pre" - Le hook d'exécution sera exécuté avant l'action spécifiée.
* « post » : le hook d’exécution sera lancé après l’action spécifiée. Les valeurs pour l’action « snapshot » sont « pre » et « post ». Les valeurs pour l’action « backup » sont « pre » et « post ». La valeur pour l’action « restore » est « post ». La valeur pour l’action « failover » est « post ».




| hookSourceID | chaîne | Vrai  a| 
Chaîne JSON contenant l'identifiant de la source du hook exécuté par ce hook. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| arguments | tableau[chaîne] | Faux  a| 
Tableau JSON de chaînes JSON pour les arguments de la source. Les valeurs définies sont :

* Nombre minimum d’articles : 0
* Nombre maximum d'articles : 16
* Longueur minimale : 0
* Longueur maximale : 127




| ID de l'application | chaîne | Faux  a| 
Chaîne JSON contenant l'identifiant associé de l'application à laquelle ce hook est lié. En essence, d'autres critères de correspondance, si définis. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| activé | chaîne | Vrai  a| 
Chaîne JSON indiquant si le hook d'exécution est activé. Les valeurs définies sont :

* "true" - Le hook d'exécution est activé et sera exécuté comme spécifié.
* « false » - Le hook d’exécution est désactivé et ne sera jamais exécuté. La valeur par défaut est « true ».




| description | chaîne | Faux  a| 
Chaîne JSON contenant une description, spécifiée par l'utilisateur, de la source du hook. Les valeurs définies sont :

* Longueur minimale : 0
* Longueur maximale : 511




| les métadonnées | link:#type_astra_metadata_update["type_astra_metadata_update"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra. Si aucune étiquette n'est spécifiée lors de la création, un objet de métadonnées sera créé sans étiquette. Si aucune étiquette n'est spécifiée lors de la mise à jour, les étiquettes de l'objet de métadonnées, creationTimestamp et createdBy, seront conservées sans modification.
