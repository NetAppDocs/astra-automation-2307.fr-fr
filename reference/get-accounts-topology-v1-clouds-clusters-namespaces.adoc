= Lister les espaces de noms d'un cloud
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/clouds/{cloud_id}/clusters/{cluster_id}/namespaces`#

Renvoie un tableau JSON contenant toutes les ressources correspondantes de la collection. Si des champs individuels sont spécifiés dans la requête, le tableau JSON items contiendra les champs demandés pour chaque ressource correspondante dans l'ordre indiqué.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| cloud_id | chaîne | chemin | Vrai  a| 
ID de la ressource cloud



| cluster_id | chaîne | chemin | Vrai  a| 
ID de la ressource de cluster contenant



| inclure | chaîne | requête | Faux  a| 
Indique les champs qui doivent être renvoyés lors de l’énumération d’une collection.

Exemples d'URL : « include=id », « include=id,name »

Exemples OpenAPI : « id »

* exemple : id, nom,kubernetesLabels




| limite | chaîne | requête | Faux  a| 
Indique le nombre maximal de ressources à renvoyer lors de l’énumération d’une collection.

Exemples d'URL : « limit=2 »

Exemples OpenAPI : « 2 »



| filtre | chaîne | requête | Faux  a| 
Indique les champs qui doivent être mis en correspondance pour renvoyer une ressource lors du listage d'une collection.

Les opérateurs pris en charge sont :

* eq - Égalité
* lt/gt - Inférieur à/Supérieur à
* LTE/GTE - Inférieur ou égal à/Supérieur ou égal à


Exemple d'URL : « filter=field%20eq%20%27value%27 »

Exemples OpenAPI : « field eq 'value' »

|===

Liens décrivant les opérations prises en charge sur la ressource.

Chaîne JSON contenant une valeur indiquant le nom de l'espace de noms. Les valeurs définies sont :

* Longueur minimale : 1
* Longueur maximale : 255




| namespaceState | chaîne | Vrai  a| 
Chaîne JSON contenant une valeur indiquant l'état de l'espace de noms. Les valeurs définies sont :

* « découvert » - L'espace de noms a été trouvé dans le cluster
* « supprimé » - L'espace de noms a été supprimé du cluster, ou le cluster n'est pas géré mais il peut exister des ressources faisant référence à l'espace de noms




| namespaceStateDetails | tableau[link:#type_astra_stateDetail["type_astra_stateDetail"]] | Vrai  a| 
Tableau JSON d'objets JSON contenant des informations supplémentaires sur la raison de la transition de l'état « state » vers « state a », « state b » ou « state c ». Si aucun détail n'est disponible, ce tableau sera vide. Les valeurs définies sont :

* Nombre minimum d’articles : 0




| kubernetesLabels | tableau[link:#type_astra_label["type_astra_label"]] | Faux  a| 
Tableau JSON d'objets JSON, chacun contenant une étiquette Kubernetes associée à l'espace de noms. Les valeurs définies sont :

* Conforme au schéma de tableau d'étiquettes Kubernetes




| clusterID | chaîne | Vrai  a| 
Chaîne JSON contenant un identifiant unique global du cluster dans lequel se trouve l'espace de noms. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| systemType | chaîne | Faux  a| 
Chaîne JSON indiquant que l'espace de noms est un type d'espace de noms système. Ce champ identifie le type d'espace de noms système, le cas échéant (version 1.1 uniquement). Les valeurs définies sont :

* « kubernetes » - Un espace de noms système Kubernetes (K8s) générique, par exemple « kube-system »
* « netapp » - Un espace de noms système NetApp, par exemple « trident » ou « netapp-monitoring »
* « openshift » - Un espace de noms système OpenShift, par exemple, « openshift-monitoring »
* "rke" - Un espace de noms Rancher, par exemple "cattle-logging"
* « autres » - Espaces de noms système courants qui ne correspondent pas nécessairement à d'autres catégories, par exemple « ingress-nginx », « tekton-system »
