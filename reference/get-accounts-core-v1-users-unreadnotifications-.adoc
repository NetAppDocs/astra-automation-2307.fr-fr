= Récupérer une notification non lue
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/users/{user_id}/unreadNotifications/{unreadNotification_id}`#

Renvoie la ressource API de notification non lue stockée dans le corps de la réponse JSON.



== Paramètres

[cols="5*"]
|===
| Nom | Type | Dans | Obligatoire | Description 


| identifiant_compte | chaîne | chemin | Vrai  a| 
ID de la ressource de compte conteneur

* exemple: {{.Account}}




| ID de l'utilisateur | chaîne | chemin | Vrai  a| 
ID de la ressource utilisateur contenant



| id de notification non lue | chaîne | chemin | Vrai  a| 
ID de la collection unreadNotification à lister

|===


== Réponse

[source]
----
Status: 200, Returns the stored Unread Notification API resource in the JSON response body.
----
[cols="4*"]
|===
| Nom | Type | Obligatoire | Description 


| type | chaîne | Vrai  a| 
Type de média de la ressource. Les valeurs définies sont :

* "application/astra-unreadNotification"




| version | chaîne | Vrai  a| 
Version de la ressource. Les valeurs définies sont :

* "1,0"




| id | chaîne | Vrai  a| 
Identifiant unique global de la ressource. Les valeurs définies sont :

* Conforme au schéma UUIDv4




| ID de notification | chaîne | Vrai  a| 
Chaîne JSON contenant un identifiant de notification à marquer comme lue par l'utilisateur créant le marquage. Les valeurs définies sont :

* Conforme au schéma d'identifiant Astra




| sequenceCount | nombre | Vrai  a| 
Numéro JSON contenant un nombre de séquences associé à la notification spécifiée par l’ID de notification.



| gravité | chaîne | Vrai  a| 
Chaîne JSON contenant l'une des valeurs suivantes indiquant la gravité de l'événement. Les valeurs définies sont :

* « Résolu » : indique qu’un événement précédemment signalé décrivant une condition n’est plus actif. Un événement dont la gravité est « Résolu » utilisera le même résumé et le même identifiant de corrélation pour permettre son association.
* « Indéterminé » - Indique que la gravité ne peut être déterminée
* « informationnel » - Indique un changement d'état à la fois attendu et normal (non défini dans X.733)
* « warning » : indique la détection d’un défaut potentiel ou imminent susceptible d’affecter le service avant que des effets significatifs ne se fassent sentir
* « critical » - Indique qu’un problème affectant le service s’est produit et qu’une action corrective immédiate est requise




| les métadonnées | link:#type_astra_metadata["type_astra_metadata"] | Vrai  a| 
Métadonnées spécifiques au client et au service associées à la ressource. Les valeurs définies sont :

* Conforme au schéma de métadonnées Astra
